<h1>Scrum poročilo</h1>

<h2>Osnovni podatki</h2>
<ul>
    <li><strong>Projekt:</strong> TO-DO aplikacija</li>
    <li><strong>Sprint:</strong> Sprint 1</li>
    <li><strong>Metodologija:</strong> Scrum</li>
</ul>

<h2>Osnovni podatki</h2>

Cilj sprinta je bil razširiti funkcionalnosti TO-DO aplikacije, izboljšati izračune statistike ter dodati podporo za slike/datoteke pri posameznih opravilih.

<h2>Pregled nalog</h2>

<h3>Zaključene naloge ✅</h3>
<ul>
    <li><strong>Razširitev modela opravila</strong></li>
    <p>Model opravila je bil razširjen z dodatnimi atributi, ki omogočajo naprednejšo obdelavo nalog in podporo nadaljnjim funkcionalnostim.</p>
    <li><strong>Logika za izračun povprečnega trajanja nalog</strong></li>
    <p>Implementiran je bil izračun povprečnega časa trajanja opravljenih nalog, kar omogoča analizo produktivnosti uporabnika.</p>
    <li><strong>Logika za izračun deleža dokončanih nalog</strong></li>
    <p>Dodan je bil izračun odstotka dokončanih nalog glede na vse naloge.</p>
    <li><strong>Prikaz statistike v uporabniškem vmesniku</strong></li>
    <p>Statistični podatki so prikazani v UI in uporabniku omogočajo hiter pregled nad napredkom.</p>
    <li><strong>Ureditev zaledja za podporo slik/datotek</strong></li>
    <p>Zaledni del aplikacije je bil razširjen za nalaganje in obdelavo slik oziroma datotek.</p>
    <li><strong>Ureditev podatkovne baze za podporo slik/elementov</strong></li>
    <p>Podatkovna baza je bila prilagojena za shranjevanje metapodatkov o slikah in njihovo povezavo z opravili.</p>
    <li><strong>Prikaz slik pri posameznem opravilu</strong></li>
    <p>Uporabniški vmesnik omogoča prikaz slike, vezane na posamezno opravilo, kar izboljša preglednost.</p>
</ul>

<h2>Ovire in izzivi</h2>
<ul>    
    <li>Sinhronizacija sprememb med modelom, podatkovno bazo in UI je povečala kompleksnost razvoja.</li>
    <li>Dodajanje podpore za slike je zahtevalo posege v več plasti sistema.</li>
</ul>

<h2>Kaj je šlo narobe</h2>
<ul>
    <li>Večina načrtovanih nalog je bila uspešno zaključena.
    </li>
    <li>Razvoj je potekal logično: model → logika → UI.
    </li>
    <li>Aplikacija je funkcionalno bogatejša kot pa pred sprintom.
    </li>
</ul>

<h2>Kaj bi lahko izboljšali</h2>
<ul>
    <li>Dokumentacijo bi bilo smiselno pisati sproti.
    </li>
    <li>Večje naloge bi bilo treba razdeliti na manjše, bolj obvladljive enote.
    </li>
</ul>
